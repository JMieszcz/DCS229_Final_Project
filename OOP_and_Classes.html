Object Oriented Programming & Classes

John Mieszczanski

Per Wikipedia:

Object-oriented programming (OOP) is a programming paradigm based on the concept of "objects", which can contain data and code: data in the form of fields (often known as attributes or properties), and code, in the form of procedures (often known as methods).

In languages such as Python, C++, and Java, all data items are objects, so these languages are referred to as object-oriented programming languages. In any of these languages we can create an object by defining a class (or using a built-in class/type, such as int in Python) and then assigning a variable to an instance of that class.

What is a class you might ask? Well a class is just a template/method for creating an object in any of the OOP languages already mentioned. The main features of a class are a constructor that initiates an instance of the class with the relevant data/variables, and a set of methods that defines the behavior of the class. These methods allow one to operate on an instance of the class or the entire class itself. Additionally, variables of a class (the data) are either instance variables, which are relevant to only a specific instance of the class, and class variables which are shared by all instances of a class of the same type. An example of a class is outlined in pseudocode below:

int main(){
    float human_age, dog_age;

    Clifford = Dog("Clifford", "Red", 12);
    Sally = Dog("Sally", "Brown", 5);
    print(Dog.counter);
    # Access the age instance variable from Clifford
    human_age = Clifford.get_age();
    # Use the dog_years() method to get the age in dog years
    # from Clifford
    dog_age = Clifford.dog_years();

In the above pseudocode example, we have a Dog class with three input parameters. We create an instance of this class by calling 

Clifford = Dog("Clifford", "Red", 12);
This initiates the constructor() method which assigns the three parameters to the instance variables name, color, and age which are special to the Clifford instance. We also have the class variable counter which increments by one every time we create a new instance of the Dog class. Therefore,

Clifford = Dog("Clifford", "Red", 12);
Sally = Dog("Sally", "Brown", 5);
print(Dog.counter);

prints 2 because we have created two instances. This counter class variable can be accessed by any instance of Dog, or by calling Dog.counter (i.e. directly accessing the class type). The main() function demonstrates how we can use methods on instances of Dog. For example, Clifford.get_age() accesses the age instance variable for the Clifford instance.

A slightly more complicated example of a class is outlined below. In this example, a Student class is created in Python:

class Student:
    __slots__ = ("fname", "lname", "class_")
    # class variable
    counter = 0

    def __init__(self, fname, lname, class_):
        # instance variables
        self.fname = fname
        self.lname = lname
        self.class_ = class_
        counter += 1

    # Checks if two instances of Student are equal
    def __eq__(self, other):
        return self.fname == other.fname and self.lname == other.lname

    # Allows one to print a Student instance
    def __str__(self):
        return self.fname + " " + self.lname + " " + str(self.class_)

    def setFirstName(self, fname):
        self.fname = fname

    def setLastName(self, lname):
        self.lname = lname

    def get_fname(self):
        return self.fname

    def get_lname(self):
        return self.lname

    def get_class(self):
        return self.class_
>>> john = Student("John", "Mieszczanski", 4)
>>> tom = Student("Tom", "Smith", 4)
>>> adrian = Student("Adrian", "Lemere", 3)
>>> print(adrian.__str__())
Adrian Lemere 3
>>> print(tom)
Tom Smith 4
>>> print(john)
John Mieszczanski 4
>>> adrian.setFirstName("Yo Adrian")
>>> print(adrian)
Yo Adrian Lemere 3
>>> print(tom == adrian)
False
>>> print(adrian == adrian)
True
>>> mashup = Student("Tom", "Smith", 1)
>>> print(tom == mashup)
True

In the above Python code snippet, we create a Student class. The constructor for this class is given by

def __init__(self, fname, lname, class_):
        # instance variables
        self.fname = fname
        self.lname = lname
        self.class_ = class_
        counter += 1

Every time we create a new Student instance, we initialize three instance variables from the three parameters and increment the class variable counter by one so we can track how many instances of Student we have. Each instance variable requires the prefix self, which is a feature unique to Python. Additionally, the class_ instance variable has an underscore to indicate it should not be mutable. We also define an __eq__ method that defines if two instances are equivalent or not. This method is demonstrated using the Python interpreter in the code below the class. The __str__ method defines how an instance of Student is printed. Lastly, we have methods for changing the first and last names of a given Student instance, and methods that allow us to access information about a given Student.

One might ask "Why use OOP and classes when we can just use functions?" The article If everyone hates it, why is OOP still so widely spread? helps us answer this question. The main advantage highlighted by the article is encapsulation. What is encapsulation? Well, it is essentially the idea in OOP that data and methods are encapsulated within an object and thus we can control how that data is accessed with our methods. As the article notes, this allows us to protect data that we do not want changed. The abstraction inherent to OOP also allows a user to access and manipulate data without having to know what is going on underneath the hood. This allows for people with a far greater range of abilities to write useful code. Inheritance and polymorphism are also inherent to OOP and discussed in this article, but those concepts will be covered in a separate document.
